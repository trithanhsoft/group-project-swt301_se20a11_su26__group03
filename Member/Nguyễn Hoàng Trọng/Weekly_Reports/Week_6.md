Chỉ code không thì chưa đủ, viết tài liệu mới là phần "ngốn" thời gian nhất. Tôi đã ngồi tổng hợp lại toàn bộ API Document bằng định dạng Markdown, ghi rõ Request/Response, Params và Header cho từng API để nhét vào thư mục `testing_documents/Test_Cases.md`.

## Viết báo cáo Reflection
Cuối cùng, tôi dành thời gian ngồi viết file `REFLECTION.md` và `AI_AUDIT_LOG.md` này. Mọi cảm xúc bực dọc lúc fix bug giờ đã chuyển thành niềm vui khi dự án hoàn thiện. Tôi đã học được rất nhiều về vòng đời phát triển phần mềm chuẩn ISTQB thông qua bài tập này.
